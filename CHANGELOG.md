# Changelog

User-facing changes, newest first. The web UI surfaces this through a
"What's New" modal and reads the running short-SHA as the version stamp.
Every commit that ships a user-visible change should add an entry here —
see `CLAUDE.md` for conventions.

## 2026-05-13
- **Bulk select + bulk delete + bulk retag + duplicate scan** — the sounds-options menu (☰) gains "Select multiple" and "Find duplicates" entries (admin-only). Select mode swaps the tile-tap behavior so a click marks the sound with a purple ring; a sticky toolbar above the grid shows the selected count + actions: **Select all visible**, **Clear**, **Retag…** (prompt for tags to add / remove), **Delete selected**, **Exit select**. Bulk delete archives the same way the single-sound delete does. Duplicate scan groups every audio file by (filesize, duration to 0.1s); two files matching on both are almost certainly the same audio.
- **Wake-word soundboard** — Superadmin → Voice Triggers gets a "Wake word" field. Set it to e.g. `soundboard` and saying "soundboard play airhorn" (or just "soundboard airhorn") in the bot's voice channel fires the best matching sound by display-name or filename. Reuses the existing per-speaker vosk pipeline, plays as role `user` (so admin-only / superadmin-only locks still apply), and is per-utterance deduped so a single sentence can't double-fire. Empty wake word disables.
- **TTS pronunciation overrides** — Superadmin → TTS Voices gains a "Pronunciation overrides" section: whole-word, case-insensitive find / replace pairs that are applied to every TTS request right before synth (and before expression segmentation, so emotion detection sees the rewritten text too). Use it for proper nouns, inside-joke names, and brand names the model mispronounces — e.g. `nginx` → `engine-x`. Recents + display still show the original input.
- **Vote tally transparency** — when a vote finalises, the Discord embed now includes the per-voter breakdown ("**Yes:** Alice, Bob — **No:** Carol") so it's clear who supported the outcome. Superadmin → Voting → Recent votes table renders the same breakdown under each row's outcome cell (green checkmark for yes, red cross for no). Voter detail is also persisted to `vote_events.voters_json` alongside the existing tally counts so the audit trail survives restarts.
- **Perf bundle** — event-loop lag histogram via `perf_hooks.monitorEventLoopDelay` is now exposed at `GET /api/diag/perf` (superadmin-only) alongside RSS / heap / uptime, so future regressions in the sync FS / FFI hot paths are catchable from outside. Sound-audio responses switched from `Cache-Control: no-cache` to `private, max-age=300` so the browser stops asking the server on every replay (ETag still invalidates on normalize). And the AudioMixer's per-frame `Buffer.concat` was replaced with a walking-copy into a pre-allocated frame — `O(N*FRAME_SIZE)` allocs per 20 ms tick collapsed to one allocation, biggest win when ffmpeg is emitting many small stdout chunks.
- **Security micro-bundle** — `ttsWavCache` (the short-lived freshly-synthesized TTS WAV) and `urlPreviewCache` (the URL-stream preview blob + meta) now reject cross-user reads: a guessed / leaked id returns 404 instead of letting another logged-in user fetch your in-flight clip. `/votetimeout` refuses to fire against a member who's already in a Discord timeout (was silently re-extending the timer). And the bot now wires a `guildCreate` listener so slash commands register the moment it joins a new guild instead of waiting for the next restart.
- **Voice-trigger + vote activity logs now persist across restarts** — the two superadmin activity feeds (Voice Triggers → Recent activity, Voting → Recent votes) used to live in a 50-row in-memory array that vanished on every service restart. They now write to SQLite (`voice_trigger_events` / `vote_events` tables in `stats.db`) and the GET endpoints query up to 200 most-recent rows with a `?limit=` knob. Restarts no longer wipe context.
- **`/api/sounds` cache + ETag** — the sounds list endpoint is polled every 1–10 s by every connected client and rebuilds a ~130 KB payload via N `statSync` calls + a full sounds.json parse. Now memoized for 10 s and served with an `ETag` so repeat polls land as `304 Not Modified` (no body, no rebuild). Cache is invalidated on every `saveSoundsMeta` call (rename, retag, volume, trim, delete, normalize, etc.) so user-visible state stays fresh.
- **Mobile UX polish** — every modal now uses `dvh` for max-height so the on-screen keyboard doesn't push Save/Cancel buttons off the bottom of the screen. Sound-tile action buttons (preview, edit, favorite) are now persistently visible on touch devices instead of relying on hover. Tapping a play button that's on cooldown (or otherwise disabled) now fires a toast explaining why instead of silently doing nothing — the renderer switched from native `disabled` to `aria-disabled` so the click handler can route through `playSound` and surface the reason.
- **Voice-clipping permissions** — Superadmin → Access & Limits gains a "Voice clipping" section with per-role toggles (Allow users / admins / superadmin); Superadmin → Users → per-user overrides gains a "Voice clipping enabled" boolean. Both the 📎 Clip button and the `/clip` slash command now hide / refuse when the caller's effective permission is off, and `POST /api/clip/capture`, `POST /api/clips/:id/save-as-sound`, and `GET /api/clips` all enforce the same gate. Defaults: on for user / admin / superadmin, off for guest. `/clip` resolves the Discord caller via the same linked-user pipeline as `/play`.
- **"Entrance / Exit" tab renamed to "Linked Users" + split toggle** — the Superadmin tab that holds Discord-user linking is now labeled "Linked Users" since `/play` (and future slash commands) also read from it. The previous single "Enable" checkbox has been split: **Enable Discord linking** is the master switch (controls whether linked accounts inherit roles + per-user limits for slash commands at all), and **Play entrance / exit sounds for linked users** is a sub-toggle that just gates the join/leave sound playback. Existing installs keep their previous behavior — when "linking enabled" is on, the sub-toggle defaults to on so entrance/exit still fires.
- **Toast z-index above modals** — toast notifications fired from inside a modal (e.g. "Give it a name first" while the Clips modal is open) were getting dimmed behind the backdrop. The toast container now sits at z-index 1400, above every modal overlay.
- **`/play` now respects every soundboard permission** — when the invoking Discord user is linked to a soundboard account (and the global Discord-link toggle is on), `/play` inherits that account's role and per-user override values. Otherwise it runs as a plain `user` and enforces the user-tier per-user cooldown + max-duration limits, plus the admin-only / superadmin-only playback locks, plus the single/multi-play role-hierarchy rules — the same gates `/api/play` uses on the web UI. Disabled linked accounts get a clear "your linked account is disabled" reply instead of silently playing. Failure reasons (`locked`, `superadmin-only`, `lower-role`, `voice-not-ready`, etc.) are rewritten as human-readable ephemeral replies.
- **Finish UX pass: no more native dialogs** — every remaining `confirm()` / `alert()` / `prompt()` call site (voice admin, training cancel, RVC/voice deletes, Suno discard, TTS preset rename + save, password reset, etc.) now uses the in-app `confirmModal` / `alertModal` / `promptModal` helpers. Native error popups in voice-triggers / voting / entrance-exit / save-flows now fire as toasts instead. The web UI no longer triggers a single browser-chrome dialog.
- **Sidechain ducking + auto-clip on voice triggers** — when TTS or a voice-trigger sound is playing, the mixer now ducks every other concurrent track by ~12 dB so speech / punchlines sit on top of music instead of fighting it. Superadmin → Voice Triggers gains an "Auto-clip on fire (s)" field — set to >0 and every trigger fire also captures the last N seconds of channel audio as a clip (visible in ☰ → Clips), so the surrounding context is preserved automatically.
- **`/play <sound>` slash command** — any member can play a soundboard sound from inside Discord without switching to the web UI. The `sound` field autocompletes against your library's display names and filenames as you type. Respects the same admin-only / superadmin-only playback gates as the web UI (`/play` invocations run with role `user`, so they obey the global lock toggles).
- **Clip modal polish** — the Discard confirm dialog is no longer see-through (it was using a `var(--card)` CSS variable that doesn't exist; falling through to `transparent`). Backdrop bumped to 92% opacity to fully mask the underlying Clips overlay. Clip-preview playhead now updates via `requestAnimationFrame` instead of HTML5 audio's `timeupdate` (which only fires ~4× per second), so the cursor glides smoothly across the waveform.
- **gzip + static cache + SQLite pragmas** — `compression()` middleware enabled (the ~660 KB frontend now ships ~120 KB on the wire), static assets get a 1-hour `Cache-Control`/`ETag` (forced refresh still triggered by the `/api/version` SHA poll on deploy), and the stats SQLite db sets `synchronous=NORMAL` + `wal_autocheckpoint=1000` so the WAL file doesn't grow unbounded.
- **Clip audio quality + editor** — the rolling-buffer mixer was placing each opus chunk by wall-clock arrival time, which jittered between consecutive 20 ms frames and caused samples to overlap or drop (the choppy / garbled output). Audio is now stored per-speaking-session with a single startTs anchor and contiguous PCM, so playback inside a turn is bit-for-bit gapless. The Clips modal also gets a proper waveform editor matching the sound-edit panel: per-clip canvas with min/max peaks, draggable purple start/end handles, playhead during preview, "Start: m:ss.s — End: m:ss.s" label, and Reset trim. The save-as-sound endpoint takes the precise start/end seconds; ffmpeg re-encodes the trimmed range. Nested modal (Discard) no longer bleeds through the underlying Clips overlay — confirm modals now sit at z-index 1300 with an opaque backdrop.
- **Lifecycle hardening** — voice-trigger and clip-capture `speaking.on('start')` listeners are now properly detached on teardown (slow EventEmitter leak across many rejoins), recognizer/decoder/stream locals are cleaned up if speaker setup throws before the entry is stored, `playTtsBuffer` single-play branch now has the same ffmpeg `close` self-heal as multi-play, `prematureCloseTimestamps` resets on `leaveVoiceChannel` (no false 3-in-30s trip on reconnect), AudioMixer tracks now retain a handle to their ffmpeg child and SIGKILL it on `removeTrack` / `removeAllTracks` so rapid Stop/Play storms don't pile up zombies, and `rebuildVoiceTriggerGrammar` snapshots speaker IDs before deleting (Map-during-iteration safety).
- **Clip button on the soundboard** — the Sounds card header gains a 📎 Clip button (sitting alongside TTS / URL / Song / Upload) with quick duration picks (10 s / 30 s / 1 min / 2 min) and a "Manage clips…" entry. Clicking a duration captures from the rolling buffer and auto-opens the Clips modal scrolled to the new clip. Same capture path as `/clip` in Discord, available to anyone logged in.
- **`/clip` rolling buffer + Clips modal** — the bot now keeps the last 3 minutes of voice-channel audio per active speaker in memory. Run `/clip` in Discord (defaults to 30 s, accepts 5–120) to mix it down into an MP3 you can preview, trim, name, and save to the soundboard. The hamburger menu gets a new **Clips** entry with a count badge for unsaved clips. Audio paths reuse the per-speaker capture pipeline that voice triggers already use; the rolling buffer lives in memory only and is cleared whenever the bot leaves a voice channel. Up to 30 clips are kept on disk; oldest are evicted automatically.
- **YouTube cookie session + PO-token sidecar for the URL streamer** — fixes "Failed to start stream" on most YouTube videos. YouTube's 2025 anti-scraping stack (bot-detection wall + GVS PO Token + nsig JS challenge + SABR streaming) was returning storyboard-only formats for yt-dlp; the bot wall hit ~75% of test videos and cookies alone made it *worse* by forcing the web-only client. Three pieces landed together: (1) a persistent Chromium under Xvfb on CT 109 holding a logged-in YouTube tab, with a noVNC tunnel reverse-proxied at `/admin/yt-vnc/` so the superadmin can log in once via the embedded browser; (2) `bgutil-ytdlp-pot-provider` as a `yt-pot-server.service` sidecar on 127.0.0.1:4416 that generates the PO tokens YouTube now requires; (3) Deno installed on the host so yt-dlp can solve the nsig JS challenge. Superadmin → Access & Limits gets a new "YouTube Cookie Session" panel: **Open browser** opens the embedded Chromium, **Enable cookies for yt-dlp** flips the env var live (no restart), **Test cookies** runs a real yt-dlp probe against the canary URL, and a 6-hour systemd timer reloads the YouTube tab to keep Google's session cookies refreshed.

## 2026-05-12
- **UX polish pass** — destructive actions (delete sound, delete user, delete tag, delete trigger, disable user, normalize sound) now use an in-app modal instead of the browser's native confirm box (looks trusted, works properly on mobile, theme-aware, Escape/Enter wired up). A keyboard focus indicator was restored across every interactive element (the previous `outline:none` stripping left keyboard users with no feedback). The Controls button in the top bar now shows a badge with the count of currently-active toggles plus a tooltip listing which ones are on, so you don't have to open the popover to check state. The soundboard auto-enters the mobile-grid layout on viewports ≤640 px (still toggleable from the Sounds-options menu). New `confirmModal`/`alertModal`/`promptModal` helpers will replace the remaining native dialogs in follow-up commits.
- **Security hardening pass** — passwords are now stored as `scrypt`-hashed values; existing plaintext users are migrated transparently on their next login. `SESSION_SECRET` is required (no more placeholder fallback). Session cookies are SameSite=lax + Secure-when-HTTPS, and the app trusts the Caddy proxy so client IPs can't be spoofed via the `X-Forwarded-For` header anymore. Failed logins are throttled at 10 per 10 minutes per IP. The absurd-captcha token compare is now constant-time. Suno API routes reject malformed task IDs before they can reach any filesystem path. Voice triggers play sounds as a regular `user` role instead of `superadmin`, so they now respect admin-only / superadmin-only modes. A `.dockerignore` was added so future Docker builds don't accidentally bake `.env`, `data/`, `sounds/`, or the vosk model into the image.
- **Stability sprint** — atomic JSON state writes (a crash mid-write can no longer blank `guest.json` and wipe every setting), in-memory cache for `guest.json` and `state.json` (removes ~75 sync disk reads/sec from the voice-trigger hot path), every voice-connection rejoin path (manual `/rejoin`, Disconnect handler, premature-close burst) now funnels through one cooldown + in-flight guard so the disconnect race can't tear down a freshly-joined connection, `ENOBUFS` (kernel UDP-buffer pressure) added to the benign-error allowlist so it no longer triggers a process restart, and `unhandledRejection` now re-throws like `uncaughtException` instead of being silently swallowed.
- **`/rejoin` slash command** — any member can run `/rejoin` to move the soundboard into their current voice channel, or to bounce it out and back in when it's gotten stuck. Works whether the bot was already in another channel or not connected at all. Limited to one bounce per 5 seconds (rapid back-to-back invocations leaked UDP sockets badly enough to crash the bot).
- **Voice triggers fire faster and more accurately** — recognition now matches against vosk's partial (mid-utterance) transcripts, so a trigger fires within ~moments of the word being spoken instead of waiting for the speaker to finish the sentence + an 800 ms silence tail. The activity log's transcript field upgrades to the full sentence once recognition finalises.
- **Word-boundary trigger matching** — phrases now match on whole-word boundaries instead of bare substring, so `wife` no longer fires on "delegate" or "midwife" and `bad` no longer fires inside "badger". Existing triggers keep working; configured phrases are still case-insensitive and whitespace-normalised.
- **Voice connection stability** — bumped `@discordjs/voice` to 0.19.2 (upstream fix for the recent "every DAVE packet fails to decrypt" regression that caused choppy / missed triggers) and added auto-recovery on Discord disconnects and after a burst of audio-player "Premature close" errors. Discord.js bumped to 14.26.4.
- **Quieter voice diagnostics** — per-packet `[DAVE] Failed to decrypt a packet` debug noise is rolled up into a single `decrypt failures last 30s: N` line so journald keeps months of real history instead of hours.
- **Voting toggle in the Controls menu** — the top-bar ⚙ Controls popover gains a "Voting (/votekick · /votetimeout)" switch (superadmin-only), mirrored with the toggle in Superadmin → Voting.

## 2026-05-11
- **Voice triggers** — Superadmin → Voice Triggers configures phrase-to-sound mappings; the bot listens to its current voice channel and plays the mapped sound when it hears the phrase. Speech recognition runs on-device (vosk small English model, grammar-restricted to your configured phrases), so no audio leaves the host. Each trigger has its own cooldown and an optional speaker filter so a phrase can be scoped to one specific Discord user. The model installs automatically on `scripts/update.sh` (~40 MB one-time download).
- **Bot no longer self-deafens** — voice connections now join with `selfDeaf: false` so the bot can hear the channel for voice triggers (previously joined deafened by default).
- **Global voice-trigger cooldown** — Voice Triggers tab adds a "Global cooldown" input that throttles all trigger fires combined, on top of each row's per-trigger cooldown. 0 disables.
- **Voice trigger activity log** — Voice Triggers tab gains a "Recent activity" table showing the last 50 matched-phrase events: when, speaker, phrase, transcript, sound, and status (fired vs. cooldown-skipped vs. global-cooldown-skipped). Useful for tuning cooldowns and spotting false positives.
- **Controls menu in the top bar** — the inline "Only admins / Only superadmin" toggles fold into a single ⚙ Controls button that opens a popover containing those toggles plus Multi-play and Voice activation. The Multi-play card section and the Voice Triggers tab's Voice-activation toggle still work as before; flipping any of the four stays in sync across both surfaces.
- **Voice triggers now log the full sentence** — vosk runs open-vocabulary instead of grammar-restricted so the activity log's Transcript column shows the whole heard sentence (e.g. "this song is so bad") rather than just the matched word. Trigger matching is unchanged — phrases are substring-matched against the transcript.
- **/votekick and /votetimeout slash commands** — members in the bot's voice channel can call `/votekick @user [reason]` (disconnects the target from voice) or `/votetimeout @user minutes:N [reason]` (Discord guild timeout). The vote panel shows live Yes/No tallies with buttons; the vote ends as soon as the threshold is met or Yes can no longer win, or when the window expires. Superadmin → Voting configures threshold %, vote window, minimum eligible voters, target cooldown, max timeout duration, and immune roles. **Required Discord permissions on the bot's role:** `Move Members` for kick and `Moderate Members` for timeout — if missing, votes still pass but the action will fail (visible in the in-Discord embed and the Recent votes log).
